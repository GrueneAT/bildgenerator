## Directory Purposes

**resources/js/:**
- Purpose: All application JavaScript source code organized by functionality
- Contains: 18 JavaScript modules split across main app and QR code submodule
- Key files: `main.js` (core state), `wizard.js` (navigation), `handlers.js` (event logic), `canvas-utils.js` (Fabric.js utilities)

**resources/js/qrcode/:**
- Purpose: QR code generation and integration functionality
- Contains: 4 specialized modules for QR code workflow
- Key files: `qrcode-generator.js` (creation), `qrcode-handlers.js` (UI bindings), `qrcode-wizard.js` (step management)

**resources/css/:**
- Purpose: All stylesheet source and compiled output
- Contains: Tailwind configuration source, compiled output, custom styles, font definitions
- Key files:
  - `output.css` - Compiled Tailwind CSS (generated, ~37KB)
  - `input.css` - Tailwind source with @apply directives
  - `style.css` - Custom GRÜNE brand styles (10KB, not Tailwind)
  - `fonts.css` - Custom font @font-face declarations

**resources/images/logos/:**
- Purpose: Hierarchical logo organization for GRÜNE branded materials
- Contains:
  - `gemeinden/` - Municipal logos with JSON index
  - `bundeslaender/` - State/federal logos with JSON index
  - `domains/` - Domain-specific logos with JSON index
  - `index/` - Generated JSON metadata files for search
- Pattern: Each logo directory has a corresponding `.json` file for searchable metadata (generated by `logo_json.py`)

**resources/fonts/:**
- Purpose: Custom Gotham Narrow font family files
- Contains: 4 font variants (Gotham Narrow Book, Ultra, Bold, Ultra Italic) in OTF format
- Usage: Loaded via `fonts.css` @font-face declarations, preloaded via FontFaceObserver

**vendors/:**
- Purpose: Third-party library bundles (not installed via npm, manually included)
- Contains: 8 vendor libraries bundled into production build
- Build: All vendors except jQuery concatenated into `build/vendors.min.js`
- Usage:
  - jQuery (`jquery.min.js`) - Kept separate as core dependency
  - Fabric.js - Canvas manipulation library (primary external dependency)
  - QRCode.js - QR code generation
  - FontAwesome - Icon library (CSS only in production)
  - Others: Mustache (templating), ImagesLoaded, Masonry, FontFaceObserver

**scripts/:**
- Purpose: Build automation and development tooling
- Contains: 6 utility scripts for bundling, CSS compilation, testing setup
- Key flow: `npm run build` → `build.js` → calls `build-js.js` and `build-css.js`

**tests/:**
- Purpose: Jest unit and integration tests (validates real code, not mocks)
- Contains: 6 test suites covering validation, canvas operations, logo management
- Pattern: Tests interact with actual DOM elements and real functions
- Run: `npm run test` (all), `npm run test:watch` (development)

**visual-regression/tests/:**
- Purpose: Playwright visual regression tests (screenshot comparisons)
- Contains: 14 test specs covering UI, templates, QR codes, wizards
- Organization: Tests organized by complexity (fast-tests, medium-tests, complex-tests projects in `playwright.config.js`)
- Run: `npm run test:visual` (all), `npm run test:visual-fast` (quick subset)

**e2e/:**
- Purpose: Playwright end-to-end integration tests (user workflows)
- Contains: 10 test specs covering complete user journeys
- Pattern: Tests interaction flows from start to finish (wizard → export)
- Run: `npm run test:e2e` (all E2E tests)

**build/:**
- Purpose: Production build output (generated directory)
- Contains: Optimized index.html, bundled JS (app.min.js, vendors.min.js), bundled CSS (app.min.css), static assets
- Generated by: `npm run build` or `make build`
- Purpose of separate build: Minification, bundling, asset optimization for GitHub Pages hosting

**.development/:**
- Purpose: Development planning and documentation
- Contains: Development notes, phase plans, analysis documents
- Pattern: Not committed artifacts but referenced during development

## Naming Conventions

**Files:**
- **Modules**: camelCase with hyphens for multi-word (e.g., `event-handlers.js`, `canvas-utils.js`)
- **Test files**: `.test.js` or `.spec.js` suffix (e.g., `validation.test.js`, `core-functionality.spec.js`)
- **Config files**: kebab-case or standard (e.g., `tailwind.config.js`, `jest.config.js`)
- **CSS**: camelCase input, generated output (e.g., `input.css`, `output.css`)
- **Images**: kebab-case (e.g., `gruene-logo.svg`, `Ankreuzen.png`)

**Directories:**
- **CSS-related**: `css/` (CSS only)
- **JS-related**: `js/` (JavaScript only) with subdirectories for features (e.g., `js/qrcode/`)
- **Images**: `images/` with hierarchical subdirs for categories (`logos/gemeinden/`, `logos/bundeslaender/`)
- **Test-related**: `tests/` with subdirs by type (`unit/`, `integration/`) and `visual-regression/` separate

**JavaScript Variables & Functions:**
- **Global state**: camelCase lowercase (canvas, contentImage, currentStep)
- **Functions**: camelCase lowercase (replaceCanvas, setValue, addLogo)
- **Objects/Modules**: PascalCase or camelCase (TemplateConstants, AppConstants, LogoState, EventHandlerUtils, CanvasUtils)
- **Constants**: UPPER_SNAKE_CASE or PascalCase in objects (AppConstants.COLORS.BACKGROUND_PRIMARY)
- **jQuery selections**: Prefixed with `$` (e.g., `$container`, `$textMethods`)

**CSS Classes:**
- **Tailwind**: Standard Tailwind utility classes (bg-gruene-primary, text-white, px-4, py-2)
- **Custom**: kebab-case (e.g., `.text-method`, `.align`, `.edit-btn`, `.fabric-canvas-wrapper`)
- **GRÜNE brand**: gruene-primary, gruene-secondary, gruene-dark (defined in tailwind.config.js)

## Where to Add New Code

**New Feature (Canvas-Related):**
- Primary code: `resources/js/main.js` (state + main functions) or `resources/js/canvas-utils.js` (if utility)
- Event handlers: `resources/js/event-handlers.js` (add to EventHandlerUtils.setupMainAppHandlers)
- Constants: `resources/js/constants.js` (add to AppConstants if configurable)
- Tests: `tests/integration/` (integration tests) and `visual-regression/tests/` (visual tests)

**New Wizard Step or UI Page:**
- Navigation: `resources/js/wizard.js` (add goToStep() branch and button bindings)
- Validation: `resources/js/wizard.js` (add validateStepN function)
- HTML: `index.html` (add step div with id `step-N`)
- Tests: `visual-regression/tests/wizard-navigation.spec.js` (add screenshot test)

**New Component/Module:**
- Implementation: `resources/js/[module-name].js` (new file in resources/js/)
- Export pattern: Global object or directly assigned to window (no ES6 modules)
- Event binding: Add to `resources/js/event-handlers.js` if it needs UI interaction
- Tests: `tests/integration/[module-name].test.js`

**Utilities/Helpers:**
- Shared utilities: `resources/js/helpers.js` (general-purpose functions)
- Canvas utilities: `resources/js/canvas-utils.js` (Fabric.js-specific operations)
- Validation: `resources/js/validation.js` (input validation rules)
- Constants: `resources/js/constants.js` (all magic numbers and configurations)

**Styling:**
- Tailwind utilities: Use existing Tailwind classes in HTML, no CSS changes needed
- Custom styles: `resources/css/style.css` (only for non-Tailwind styles)
- Theme colors: Modify `tailwind.config.js` if brand colors change
- Fonts: Add @font-face to `resources/css/fonts.css`

**Tests:**
- Unit tests: `tests/unit/[module-name].test.js` (test isolated functions)
- Integration tests: `tests/integration/[module-name].test.js` (test interactions)
- Visual tests: `visual-regression/tests/[feature].spec.js` (add to playwright.config.js testMatch)
- E2E tests: `e2e/[workflow].spec.js` (test complete user journeys)

## Special Directories

**build/:**
- Purpose: Production-optimized output directory
- Generated: Yes (created by `npm run build`)
- Committed: No (added to .gitignore)
- Contents: Minified HTML, bundled JS (app.min.js + vendors.min.js), bundled CSS (app.min.css), static assets
- Purpose: Deployed to GitHub Pages hosting
- Refresh: Run `npm run build` after code changes before deployment

**visual-regression/reference-images/:**
- Purpose: Baseline images for visual regression test comparisons
- Generated: Via `npm run generate-references` (or manually updated)
- Committed: Yes (version controlled for test stability)
- Pattern: Each test creates a -reference.png file; tests compare against this baseline
- Update: When intentional UI changes are made, regenerate references

**visual-regression/comparison-results/:**
- Purpose: Generated diffs and comparison images from failed tests
- Generated: Automatically by Playwright on visual test failure
- Committed: No (temporary test artifacts)
- Pattern: -comparison.png (actual), -diff.png (visual diff), -reference.png (expected)

**coverage/:**
- Purpose: Jest code coverage reports (generated)
- Generated: Via `npm run test:coverage`
- Committed: No (build artifact)
- Contents: HTML and JSON coverage reports showing test coverage by file and line

**dist/ and e2e/:**
- Purpose: Legacy test output directories
- Status: Not actively used in current pipeline
- Note: Visual regression and E2E tests use modern Playwright under test:e2e and test:visual commands
